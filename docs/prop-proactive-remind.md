# 企画書：プロアクティブ・コンテキスト・リマインダー (Zero-Query Guidance)

## 1. 背景・課題
現在のAIエージェントの多くは、ユーザーからの命令を受けてから情報を検索（RAG）します。しかし、開発中に一番恐ろしいのは「エージェントが過去の失敗を忘れていることに、ユーザーが気づかずに指示を出してしまう」状況です。
- **検索の漏れ**: 適切な検索ワードをエージェントが選べず、関連する過去の教訓をスルーしてしまう。
- **後付けの指摘**: コードを書き終わった後にレビューエージェントに指摘されるのは二度手間であり、時間的損失が大きい。
- **認知負荷**: ユーザーが常に「過去の失敗」を意識して、プロンプトに防衛策を含める苦労。

## 2. ソリューション：Gnosis Proactive Remind
エージェントが命令を受ける前、あるいは思考を開始した瞬間に、現在のファイル・位置・コードの断片から「関連性が極めて高い過去の教訓」をGnosisが能動的にプッシュします。

### 主な機能
- **Intention Watcher**: エージェントが読み込んでいるファイルとカーソル位置から、現在の「実装意図」をリアルタイムで推論。
- **Pre-emptive Injection**: `search_memory` が呼ばれる前に、関連性の高い `Guide` や `Lesson` をエージェントのシステムプロンプトへ強制的に注入。
- **Similarity Alert**: 「過去にこのファイルで発生したバグに近いコード」をエージェントが書こうとした瞬間、ユーザーとエージェントの両方に警告を発する。

## 3. 期待される効果
- **「二度と同じ地雷を踏まない」の自動化**: エージェントの検索能力に依存せず、過去の知見を確実に再利用。
- **開発スピードの向上**: リワークを最小限に抑え、最初から正しい（規約や教訓に沿った）コードを生成。
- **能動的なAIパートナー**: 受動的なツールから、設計の不備を先回りして指摘する「ベテランのペアプロ相手」への進化。

## 4. 実装ロードマップ
- **Phase 1**: ファイルオープン・カーソル移動等のイベントに基づいた、Gnosis内部のプレビュー検索機能。
- **Phase 2**: MCPのプロトコルを通じた、エージェントへの「非同期通知・注入」インターフェースの確立。
- **Phase 3**: ユーザーがリマインドの適合度（ノイズか有用か）をワンクリックで評価する学習機能。

## 5. 想定される課題
- **ノイズ化の回避**: 些細なリマインドが多すぎると開発を阻害する（確信度スコアによる厳格なフィルタリングが必要）。
- **コンテキスト容量の管理**: 注入する情報がエージェントのトークン制限を圧迫しないような、高度な要約技術。
