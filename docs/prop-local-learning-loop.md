# 企画書：ローカル成果観測による学習ループ (Local Outcome-Aware Learning Loop)

## 1. 背景・課題
AIエージェントによる作業が「完了」した際、その成果が本当に正しかったかどうかを客観的に判断する仕組みが必要です。しかし、Gnosisはローカル開発環境での利用を主眼としており、外部のCIサーバー等に依存することはセキュリティやプライバシーの観点から望ましくありません。
- **偽の成功**: エージェントが「修正しました」と言っても、ローカルでのビルドやテストを通していない場合、潜在的なバグが残る。
- **フィードバックの欠如**: ローカルテスト (`npm test` 等) で失敗した際、その詳細なスタックトレースやログが「失敗体験」として自動的に蓄積されない。
- **セキュリティ・プライバシー**: クラウドCIサービスとの連携は、重要なソースコードやログが外部に露出するリスクを伴う。

## 2. ソリューション：Local Discovery & Feedback
Gnosisが「開発者の手元」で実行されるテストコマンドやビルド結果を直接観測し、知識グラフの信頼度スコアを安全に更新します。

### 主な機能
- **Local Command Observer**: ターミナルで実行される `npm test`, `pytest`, `cargo test` 等の終了ステータスと出力をキャプチャ。
- **Local Git Hook Integration**: `pre-commit` や `post-commit` フックと連携し、コミット前の検証結果を自動的にメモとして記録。
- **Offline Reliability Scoring**: 外部サーバーを介さず、ローカルDBのみで `Procedure`（手順）や `Entity`（知識）の成功率を計算・更新。

## 3. 期待される効果
- **「動くコード」に基づいた確実な学習**: ユーザーの主観ではなく、実際の実行結果に基づいた高精度な知識の蓄積。
- **完全なプライバシーの維持**: すべての観測・解析・保存プロセスがローカルマシン内で完結し、外部へのデータ流出を防止。
- **開発効率の最大化**: ローカルテストが失敗した際、その原因（ログ）をエージェントが即座に「失敗事例」として学習し、次の試行で同じミスを回避。

## 4. 実装ロードマップ
- **Phase 1**: 手動実行されるテストコマンドのラッパーまたは監視ツールの開発（例: `gnosis run "npm test"`）。
- **Phase 2**: 各種言語・フレームワークの標準的なテスト出力のパーサー実装。
- **Phase 3**: ローカルGitフックとの自動連携インストーラーの提供。

## 5. 想定される課題
- **環境差異の吸収**: 開発者ごとに異なるテスト環境やエイリアスの管理。
- **ノイズの分離**: ネットワークの一時的な不安定さによる失敗など、コードの品質とは無関係な情報のインテリジェントな排除。
- **パフォーマンス**: コマンド出力の解析プロセスがローカルの開発作業を妨げないような軽量な実装。
