# 企画書：自己最適化ローカルLLM (Experience-Driven Fine-Tuning)

## 1. 背景・課題
AIエージェントの性能はモデルの「汎用知」に依存していますが、実際の開発現場では「特定のライブラリの癖」「独自のコーディング規約」「ドメイン固有の例外処理」といった「現場知」が極めて重要です。
- **指示の繰り返し**: 教えたはずのプロジェクトの約束事を、新しいチャットのたびに再説明する必要がある。
- **微調整の限界**: プロンプトエンジニアリング（RAG）だけでは、モデルの「思考の癖」そのものを変えることは難しい。
- **プライバシー**: クラウドLLMに独自の知見を学習させることは、セキュリティ上困難である。

## 2. ソリューション：Gnosis Automated Fine-Tuner
Gnosisに蓄積された `vibe_memories` (Episode) と `experience_logs` (Lessons) を元に、ローカル環境で自動的にファインチューニング用データセットを生成し、ローカルLLMを最適化します。

### 主な機能
- **Dataset Synthesis**: 成功したエピソードを「Q&Aペア」や「Instructionペア」に変換。
- **LoRA Automatic Trainer**: バックグラウンドで軽量なLoRA（Low-Rank Adaptation）学習を実行。
- **Model Hot-swapping**: 学習済みの「プロジェクト特化型LoRA」を作業に合わせて動的にロード。

## 3. 期待される効果
- **「言わなくてもわかる」の実現**: 何度も説明することなく、プロジェクトの意向を汲み取ったコード生成が可能になる。
- **精度の向上**: 過去の失敗体験をモデルの重みレベルで回避するため、同様のバグの発生を物理的に抑制。
- **真のローカル特化**: 外部に一切データを出すことなく、自分たちだけの最強のジュニアエンジニア（AI）を育成できる。

## 4. 実装ロードマップ
- **Phase 1**: 教師データの抽出・クレンジングツールの開発。
- **Phase 2**: `MemoryLoopRouter` と連携した学習完了後のLoRAアダプタの運用開始。
- **Phase 3**: ユーザーのフィードバックに基づいて、学習の「強さ」を自動調整する仕組みの実装。

## 5. 想定される課題
- **学習データの品質**: 誤った「教訓」を学習してしまうリスクへの対策（Confidence値による厳選）。
- **計算リソース**: バックグラウンドでの学習実行時におけるCPU/GPU負荷の管理。
