# Gnosis Supervisor 導入計画書 (Implementation Plan)

## 1. 背景と目的
現在、Gnosis はプロジェクトごとに個別の MCP サーバー（Node.js/Bun プロセス）を起動する「プロジェクト・ローカル型」の構成をとっています。これにより、以下の問題が発生しています：

- **ゾンビプロセスの蓄積**: IDE (Cursor等) がクラッシュしたり通信が途絶えたりした際、後処理（DB切断等）が行われずプロセスが残り続ける。
- **リソースの重複消費**: 同一ユーザーが複数のプロジェクトを開くと、その数だけ PostgreSQL 接続プールやキャッシュメモリが消費される。
- **排他制御の複雑化**: 同一 DB に対して複数のプロセスが競合し、マイグレーションや書き込み時にデッドロックのリスクがある。

本計画では、これらを解決するために **Gnosis Supervisor (デーモン)** 方式への移行を提案します。

## 2. ターゲットアーキテクチャ
「1 ユーザーあたり 1 デーモン」の構成へ移行します。

### 構成要素
1.  **Gnosis Supervisor (Master Daemon)**:
    - ユーザーログイン時に `launchd` で自動起動。
    - データベース接続プールを一元管理。
    - Unix Domain Socket または localhost ポートで待機。
2.  **Gnosis Client (MCP Adapter)**:
    - IDE 側が起動する軽量なプロキシ。
    - Supervisor に接続し、コマンドを転送するだけ。
    - Supervisor からのハートビートに応答し、切断されたら即座に終了する。
3.  **Task Workers (Ephemeral)**:
    - 重い計算やバッチ処理（埋め込み、知識抽出等）を行うための、Supervisor 配下の子プロセス。

## 3. 「肥大化」への対策 (Anti-Bloat Strategy)
デーモン化によるメモリ肥大化を防ぐため、以下のガードレールを設けます。

- **アイドルタイムアウト**: 
  - 一定時間（例: 30分）ツール呼び出しがない場合、重いキャッシュやコネクションプールを縮小または開放する。
  - すべてのクライアントが切断されてから一定時間後に、デーモン自体を休止状態にする（または自動終了し、次のアクセスで再起動させる）。
- **メモリ・リサイクル**:
  - `global.gc()` を活用し、定期的にガベージコレクションを明示的に要求する（Bun/Node の `--expose-gc` 利用）。
  - Worker プロセスを「使い捨て」にし、1タスクごとに終了させることで、ワーカー側でのメモリリークを完全に封じ込める。
- **リソースクォータ**:
  - 全体で使用する PostgreSQL 接続数の上限をデーモン側で厳格に管理する。

## 4. プロセス管理フロー (Lifecycle Management)
1.  **起動**: IDE が MCP サーバーを要求。
2.  **接続**: Client が Supervisor に接続。
3.  **登録**: Supervisor が Client の PID と接続情報をレジストリに登録。
4.  **監視**: Supervisor が Client の `stdin/stdout` およびプロセス存在を常時監視。
5.  **終了**:
    - 正常終了: Client が終了を通知。
    - 異常終了: Supervisor が Client の死を検知し、関連する Worker とリソース（DB接続等）を強制解放。

## 5. 実装フェーズ

### Phase 1: 守護プロセス (Guardian) モード
- 現行の `src/index.ts` に「親プロセス監視 (Watchdog)」を追加。
- 親が死んだら自決する仕組みを徹底。

### Phase 2: Supervisor 試作
- `src/supervisor/` の作成。
- Unix Socket 通信の実装。
- DB コネクション共有の検証。

### Phase 3: ユーザー環境への展開
- `launchd` 用の `.plist` 自動生成と登録スクリプトの整備。

## 6. 成功指標
- 1日稼働後の迷子プロセス数が 0 であること。
- 複数プロジェクト起動時のメモリ消費量が、現行の 1/n 以下になること。
- デーモン自体のメモリ使用率が、アイドル時に一定値（例: 50MB以下）を維持すること。
