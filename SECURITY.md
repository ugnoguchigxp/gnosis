# Security Policy

Gnosis の脆弱性報告ポリシーです。

## 対象範囲

主に以下を対象とします。

- MCP サーバーとそのツール
- KnowFlow、Guidance、ログ同期処理
- Monitor アプリ
- セットアップスクリプト、DB 初期化処理
- 認証情報、シークレット、ローカル通信に関わる実装

## 報告してほしい内容

- 任意コード実行
- 権限昇格
- シークレット漏えい
- 想定外の外部公開
- ローカルファイルの不適切な取り扱い
- XSS、コマンドインジェクション、SQL インジェクション
- WebSocket やローカル API の不適切な公開

## 報告方法

脆弱性と思われる内容は、公開 Issue では報告しないでください。

次のいずれかの非公開手段を利用してください。

1. GitHub の private vulnerability reporting が有効なら、それを利用する
2. 既に共有済みのメンテナー向け非公開連絡先がある場合は、そこに連絡する

## 報告時に含めてほしい内容

- 影響範囲
- 再現手順
- PoC があればその内容
- 想定されるリスク
- 回避策があればその案

## 初動方針

- まず再現確認を行う
- 影響が大きい場合は修正を優先する
- 公開 Issue や公開 PR での詳細共有は、修正または緩和後に行う

## 注意

このリポジトリはローカル実行を前提としていますが、ローカル専用だから安全とは限りません。外部公開、localhost バインド、シークレットフィルタ、ファイル取り込みなどは継続的に注意してください。
